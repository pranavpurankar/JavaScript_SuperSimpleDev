What is an object

An objects groups multiple values together. Syntax: object{ property_1: value_1, property_2: value_2 }

Why we use objects?

Instead of creating bunch of variable like name, price, and other things. This will make our code lot cleaner.

Built in Objects

JavaScript has built-in object like we've seen console. There are other objects also. Two more built-in objects JSON and localstorage, we'll use those objects to improve our rock, paper, scissors project.
  • JSON

    • Helps is in JavaScript Object Notation. It is basically a syntax similiar to JavaScript objects but has less feature. All properties and string must use JSON, does not support single quote and functions.
    • Why would we use JSON syntax? JavaScript object syntax only makes sense in javascript. So to talk to or to share information other programming languag, we use JSON.
    JSON.stringify => returns a string. This JSON string ready to be sent to another computer or ready to be save in storage.

  • Local Storage

    • Local storage used to save values more permanently. However variables are temporary.
    localStorage => is object setItem => is method
    • null vs undefined
    null=> intentionally want something to be empty.
    undefined =>a variable has been declared but has not yet been assigned a value

  • Auto=Boxing

    • JavaScript automatically wraps a string in a special object.
    Ex: console.log('hello'.length) => returns lenght of string
    console.log('hello'.toUpperCase()); => return uppercase hello
    So 'hello' is wrapped in the special object; javascript does this automatically this is know as auto-boxing.
    Auto-Boxing is works with numbers and booleans but does not work with null and undefined.

    The actual value is stored in the memory, and object is reference to this memory location. This is like a shortcut on our computer system, it points to the actual memory location where it is stored. I already know about this.

This is because eventhough const is preventing us from changing the reference the variable, but it doesn't prevent us from reaching into the reference and changing the object that is pointing to.
